# req__RequirementStatusActive

Active — the requirement is **in force**. It has been approved, implemented, and merged, and its revert-verified test passes in CI. The CI traceability gate hard-enforces this: an `active` requirement that loses its passing binding (the test goes red, or the `@req` binding is removed → orphan/dangling) turns CI **red**. The only way to stop enforcing an active requirement without fixing the code is to explicitly move it to `Deprecated` (a conscious withdrawal). This realizes the invariant: *an active requirement is either satisfied (CI green) or explicitly withdrawn.*
