# req__RequirementBindingClassUiAcceptance

Manual UI acceptance via computer-control (live Obsidian / real desktop UI). A
verification tier parallel to the automated `unit`/`integration`/`e2e`/`gui-bdd`
jest bindings: such requirements are verified by **computer-control evidence**
(screenshot + date + observed result), **NOT** by a `@req:<uid>` jest tag in a
test title. The traceability checker therefore does **not** expect a jest
binding for a `ui-acceptance` requirement and does **not** flag it as an orphan
for missing one. Real-prod-exercising (live UI) → **satisfies the P0
binding-class floor** alone. (RFC 0003 §3.6 verification-tier extension.)
