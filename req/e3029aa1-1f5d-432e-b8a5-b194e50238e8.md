# req__Requirement_evidence

Committed evidence artifact(s) backing a `ui-acceptance` manual verification binding (RFC 0003 §3.6 evidence-attestation). String form `<path>` — a repo-relative path to a committed screenshot or attestation `.md`, resolved by the traceability checker relative to the requirement asset's own directory. Makes "recorded evidence" mean **committed + linked + CI-resolved**, not "trust the prose". Cardinality 0..N.

The `exocortex requirements audit` checker enforces, without running a GUI: a declared `_evidence` path that does not resolve to a committed file is a HARD dangling-evidence finding (symmetric with a dangling `@req` tag); and an `active` `ui-acceptance` requirement with no resolving evidence is an active-requirement invariant violation (the always-on blocking gate). Proposed/Draft ui-acceptance reqs are reported but not gated.
